Teletician

A tangible deployment to support personal fruit-intake habits in the home-based cardiac rehabilitation context.

This is collaborated project with Maxima Medisch Centrum in the Netherlands in an e-commerce domain. The final deliverable of the project is an ioT system that includes a tangible design and the connected digital UI design. The business values within the scope was to identify the touchpoints between the patients and the invloved healthcares, providing a design solution that can implement to the user journey.

The designed system is aimed to facilitate cardiac patient’s personal fruit-intake behaviors in the home-based remote rehabilitation context, allowing seamless integration with existing workflows and infrastructure.

This entire project mainly went through 3 full iterative cycles.The first iteration aimed to explore the remote rehabilitation context and the potential end-user’s experiences. And second and third focused on the (digital) product development. Within each iteration, I demonstrated design techniques like user research, user flow analysis, interative (functional) prototyping, iterative solution development and usability testing. Together with (hardware and software) developers, I experimented with the complex front-end and back-end systems to implement the final design in the context of iterative user evaluation.

At the end of the project, I made use of the user journey map to propose the way that could include the design to the current remote cardiac rehabilitation user flow and the Platform infrastructure. The findings provide guidelines for designing interaction between user’s eating behaviors and home system performance, also demonstrated how user’s behavior data can be communicated between hardware and software.

Time frame: 03.2022 - 05.2022
Client: Maxima Medisch Centrum
Techniques/Tools: Arduino(hardware development), figma(UI development), Rhino (CAD), 3D print, Usability Testing, Data Analysis

Service Design & Patient/User Journey  

A service blueprint is therefore built to visualize the patient/ user journey with the interactions among stakeholders and services relevant to the design. The patient is introduced to cardiac rehabilitation (CR), where he or she learns about the importance of maintaining personal health. Especially following a healthy eating pattern is emphasized. Therefore, specific diet goals are identified for the patient to follow during the CR program. Having regular fruit intake, for example, is one of the prioritized goals for the patient. The patient is introduced to the system at the start of the CR program. For first-time use, there will be a greeting message sent from an app to guide the user through the first set according to the patient’s identified goal, and then mapping the device. For everyday use, a series of looping behaviors is provided by an interactive fruit bowl to support the patient’s desired lifestyle, at the same time, collecting the patient’s performed behaviors. A connected application is used to evaluate the patient’s contextual situation, visualize the user’s performance in relation to the target goal, and even provide feedback and suggestions.
